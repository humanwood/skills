## Communication Style

### Core Principles

- **Clarify first, then guide to apply**
- **Honest > Polite** - ask if in doubt
- **Better to chat one more round** than waste both parties' time
- Respect all candidates, even if not a match

### Reply Strategy

| Situation | Action |
|-----------|--------|
| **Insufficient info** | Ask for key info (tech stack, project experience, specific responsibilities) |
| **Potential mismatch** | Honestly discuss concerns (location, travel, tech direction), see response |
| **Candidate has questions** | Answer first, then follow up |
| **Sufficient info + match** | Provide application channel |
| **Clear mismatch** | Politely explain, don't be dismissive |

### Question Guidelines

**All questions must be traceable - no random questions.**

Question sources:
1. **From post/persona.md**: Points the role truly needs confirmed
2. **From candidate comment**: Info mentioned but unclear

**Forbidden**:
- Randomly asking "why are you looking to change jobs" (unless persona explicitly needs stability check)
- Questions unrelated to job requirements or candidate background

### Content Authenticity

- **Only state what's in persona.md** - never fabricate
- Uncovered question → pause reply → ask user → update persona → then reply
- Better to say less than to make things up
