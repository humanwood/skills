# 产品文档与 API 文档评审 Rubric

## 评分维度（总分 100）

- 文档可发现性（20）
- 文档结构完整性（25）
- API 设计一致性（20）
- API 可用性（20）
- 变更与可运维性（15）

## 文档结构检查点

- 是否有产品总览（Overview）
- 是否有快速开始（Quick Start）
- 是否有开发者参考（Developer Reference / API）
- 是否有 SDK 或 CLI 指引
- 是否有错误码/常见问题/排障章节

## API 文档检查点

- 命名是否统一（动词 + 资源）
- 是否存在大量过时 API（deprecated）
- 参数描述是否清晰，示例是否充分
- 错误码是否覆盖核心失败场景
- 响应结构是否具有稳定字段约定

## 建议分级

- P0: 高风险，影响上线或误调用（缺少关键参数说明/认证方式不清晰）
- P1: 中风险，影响开发效率（缺少示例、错误码场景不足）
- P2: 低风险，体验优化（目录组织、交叉链接、最佳实践补充）
